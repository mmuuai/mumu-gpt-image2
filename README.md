# 玩AI的木木-GPT-Image2图像生成

ComfyUI 自定义节点，调用 OpenAI 兼容的 `gpt-image-2` 图像接口，支持文生图和图生图。

## 安装

把本目录复制到 ComfyUI 的 `custom_nodes/ComfyUI-Mumu-GPT-Image-2`，重启 ComfyUI 后在 `木木节点` 分类中添加 `玩AI的木木-GPT-Image2图像生成`。

## 配置

复制 `config.example.json` 为 `config.json`，填入：

```json
{
  "api_base": "https://aimumu.top",
  "api_key": "sk-你的密钥"
}
```

`config.json` 已被 `.gitignore` 忽略，不应提交到仓库。

## 模式

- `文生图`：调用 `/v1/images/generations`。
- `图生图`：连接 `参考图片` 或填写 `参考图片URL` 后调用 `/v1/images/edits`。
- `自动`：有参考图时自动图生图，否则文生图。

注意：aimumu 后台当前的图生图路由不接受 `n` 参数，并且非流式请求可能被长连接中断，所以节点会让图生图固定输出 1 张并自动使用流式请求。`图片数量` 主要用于文生图。

## 画面比例与清晰度

请求时会按映射表转换成 OpenAI 官方需要的具体像素尺寸：

- `画面比例` + `清晰度` → `size: WIDTHxHEIGHT`
- 例如 `16:9` + `4k` 会发送 `size: 3840x2160`
- 例如 `1:1` + `2k` 会发送 `size: 2048x2048`

`画面比例` 下拉包含官方 15 种比例：

- `auto`
- `1:1`
- `3:2`
- `2:3`
- `4:3`
- `3:4`
- `5:4`
- `4:5`
- `16:9`
- `9:16`
- `2:1`
- `1:2`
- `3:1`
- `1:3`
- `21:9`
- `9:21`

`清晰度` 支持 `1k`、`2k`、`4k`，节点会把比例和清晰度映射为官方接口要求的具体像素尺寸。

节点只保留 `图片` 输出口，隐藏调试 JSON 和保存路径输出，保持画布更清爽。

部分图生图后端会忽略请求里的 `size`，返回接近参考图比例的图片。节点默认开启 `输出匹配尺寸`，会在接口返回后按目标尺寸居中裁剪并缩放，确保 ComfyUI 最终输出和你选择的比例/清晰度一致。

## 参考图

图生图默认只显示 `参考图片1`。当 `参考图片1` 接上线后，节点会自动显示 `参考图片2`；继续连接会继续增加，最多 16 张。也支持一个图片输入里传入批量图，总数最多 16 张。

## 请求协议

`请求协议` 默认是 `自动`。节点会使用异步任务协议：提交后自动轮询任务结果，并把返回的图片 URL 下载为 ComfyUI 图片输出。本地参考图会先上传到 `/v1/uploads/images`，再作为 `image_urls` 调用生成接口。

节点界面只显示常用参数：`提示词`、`key`、`接口地址`、`画面比例`、`清晰度`、输出格式和参考图片。模型固定使用 `gpt-image-2`，质量等高级请求参数使用内置默认值，避免节点面板过长。
